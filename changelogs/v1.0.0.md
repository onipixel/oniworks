# Changelog — v1.0.0

**Released:** 2026-04-28  
**Tag:** `v1.0.0` — Initial public release of OniWorks

---

## Added

### Framework Core (`framework/`)
- `framework/app` — central `Application` struct; `Serve()`, `IsDebug()`, automatic error handler wiring
- `framework/routing` — HTTP router with route groups, URL parameters, named middleware stacks
- `framework/http` — `Context`, `Request`, `Response`, `HandlerFunc`, `MiddlewareFunc`, `UploadedFile`, `ParseUpload`
- `framework/database` — fluent query builder (`Table`, `Where`, `WhereIn`, `Join`, `LeftJoin`, `GroupBy`, `OrderBy`, `Limit`, `Offset`, `Select`, `Insert`, `Update`, `Delete`, `First`, `All`, `Count`, `Pluck`, `Raw`); `ErrNotFound` sentinel; batch eager-loading pattern
- `framework/migrations` — schema builder (`CreateTable`, `DropIfExists`, column helpers); `Migrator`; global `Register` / `LoadRegistry` for `init()`-based auto-registration
- `framework/auth` — JWT `Guard`; `IssueToken`, `ParseToken`
- `framework/realtime` — WebSocket `Hub`; per-channel pub/sub broadcast; `ServeHTTP` upgrader
- `framework/frontend` — `Manager` for Vite dev-server script injection and production manifest loading; `ViteTag()`
- `framework/errors` — debug-aware error handler; rich HTML error page with colour-coded stack traces in dev mode; verbose JSON for API callers; clean JSON for `HTTPError` types in all modes

### CLI (`cmd/oni`)
- `oni new <name>` — scaffold a new OniWorks project
- `oni serve [-p port]` — build and run the application, injecting `APP_PORT`
- `oni make:migration <name>` — generate a timestamped migration file from stub
- `oni make:model <name>` — generate a model file from stub
- `oni make:controller <name>` — generate a controller file from stub
- `oni make:channel <name>` — generate a realtime channel file from stub
- `oni migrate` — run pending migrations via `--oni-cmd` dispatch
- `oni db:create` — create the configured database
- `oni db:drop` — drop the configured database

### Stubs (`stubs/`)
- Migration, model, controller, channel, and main-go stubs with correct import aliases and interface compliance

### Documentation (`docs/`)
- `getting-started.md`, `routing.md`, `auth.md`, `database.md`, `frontend.md`, `cli.md`

## Fixed

| # | Area | Description |
|---|------|-------------|
| 1 | CLI / `oni migrate` | `--oni-cmd` flag was parsed but never dispatched; migration logic was never reached |
| 2 | CLI / `make:channel` | Stub output was written to project root instead of `app/channels/` |
| 3 | CLI / `oni serve` | Port flag was accepted but never passed to the spawned process |
| 4 | Docs | `getting-started.md` stated Go 1.22+ but `go.mod` requires 1.25 |
| 5 | Docs | `docs/routing.md` did not exist |
| 6 | Docs | `docs/auth.md` did not exist |
| 7 | Docs | `database.md` referenced `gorm.Expr` which is not part of OniWorks |
| 8 | Docs | `database.md` had an incorrect `Pluck` signature |
| 9 | Migration stub | `Up`/`Down` declared `error` returns, conflicting with the `Migration` interface |
| 10 | CLI | No `db:create` / `db:drop` commands existed |
| 11 | CLI | `make:*` commands failed inside a scaffolded app (stubs not embedded in binary) |
| 12 | Frontend | Vite `publicDir` warning during `npm run build` |
| 13 | Database | `normalizePlaceholders` called with `buildSelect()` result as a single `[]any` argument instead of spread — all parameterised queries failed with *"expected N arguments, got 1"* |
| 14 | Database | `Insert()` included `id = 0` for new records, preventing `RETURNING "id"` from populating auto-increment primary keys |
| 15 | API | `PostController.Show` called `enrichPosts` on a copy of the struct; enriched data was discarded |
| 16 | Error handling | Unhandled 500s showed only `{"error":"internal server error"}` with no diagnostic info in dev mode |
| 17 | Error handling | No dev-mode error page; stack traces were invisible |

## Notes

- PostgreSQL is the primary supported database; MySQL support is available via the CLI's `db:create` / `db:drop` commands
- The `testing/onigram` directory contains an Instagram-style reference application (OniGram) used to validate the framework during v1.0.0 development
- All issues discovered during OniGram development are logged in `testing/NOTES.md`
