# Changelogs

Each file in this folder documents changes for a specific version or milestone.

## Naming Convention

```
v{MAJOR}.{MINOR}.{PATCH}.md     — stable releases
unreleased.md                   — work in progress, not yet versioned
```

## Format

Every entry should follow this structure:

```md
## [vX.Y.Z] — YYYY-MM-DD

### Added
- New features

### Fixed
- Bug fixes

### Changed
- Modifications to existing behaviour

### Removed
- Removed features or APIs
```
